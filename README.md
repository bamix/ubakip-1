Курсовой проект: UBAKIP.RU
===========================

Используемые технологии, фреймворки:
```
- C# ASP.Net MVC 5
- MSSQL
- Entity Framework (EF)
- AngularJS
- Bootstrap
- HTML5
- Адаптивная верстка
- Lucene Search Engine (Lucen)
```
Техническое задание:
-------------------
Реализовать комикс-портал, где пользователи смогут создавать комиксы, просматривать созданные комиксы, обмениваться комиксами, выставлять рейтинг комиксу, писать комментарий к комиксу. 
Минимум кнопок.

Главная страница должна содержать:

- Последние комиксы
- Комиксы с самым высоким рейтингом
- Облако тегов

Каждый комикс должен содержать: 

- Название
- Тело комикса (Набор изображений-страниц)
- Информация о возрастном ограничении (Everyone, 13-16, ... 18+; или MPAA-система)
- Набор тегов

Должно быть три режима: режим просмотра, режим редактирования, режим создания. 

Пользователь открывает комикс в режиме просмотра. 

Режим редактирования позволяет изменить контент поста; редактирование текста осуществляется щелчком по тексту. 

Режим создания должен вызываться отдельно от режима просмотра и режима редактирования.

Создание комикса:
-------------------
Несколько шаблонных layout object-областей (минимум 4); в каждой части области можно разместить изображение, видео, текст (в виде облачка-выноски). 

Каждая область должна поддерживать drag'n'drop. 

Помещённая в область картинка или видео должны обрезаться по границам области (fitting). 

Картинку или видео в области возможно перемещать, изменать размер (слайдером). 

В области может быть только один объект типа изображение/видео; количество текстовых блоков неограничено. Текст добавляется поверх изображения/видео.

Опционально: возможность создавать пользовательские шаблоны.

Возможность изменить порядок страниц.

Реализовать:
-------------------
- Полнотекстовый поиск по всему контенту (Lucene), соответственно, индексирование текста. Полное сканирование БД на совпадения запрещено. Поиск должен быть доступен с каждой страницы (желательно расположить его в верхней части страницы (header)). Результаты поиска должны представлять список комиксов (например, первые 10 результатов).

- Локализации: RU / EN. Привязка выбранной локализацации к устройству, когда пользователь 'guest', и к user-id, когда пользователь аутенфицирован.

- Две визуальные темы: светлая и темная. Тема выбирается как незик (???).

- 3 пользовательские роли: Неаутенфицированный (guest), обычный пользователь (user), администратор (admin).

1) 'guest': Только чтение, поиск.

2) 'user': права пользоавтеля 'guest' + создание нового материала; редактирование собственного материала; написание комментариев к пользователям; выставление рейтинга.

3) 'admin': Полный доступ: чтение, создание, редактирование любого созданного материала (в том числе, и от имени пользователя); выдача наград; изменение информации о пользователе; удаление и блокировка пользователей; сбрасывание пароля пользователя.

- Система тегов. Система должна поддерживать автодополнение (при вводе начальных букв тега, выводится список всех существующий на сайте тегов по данной маске).

Панель пользователя (user-page)
-------------------
Панель пользователя содержит: 

1) Информацию о пользователе. {First name, Last Name, About yourself}

2) Список всего контента, созданного пользователем. Данный список может быть сортирован по дате создания, рейтингу, алфавиту.

3) Награды пользователя (медальки).

4) Аватар пользователя

Награды
-------------------
Должны иметь графическое представление.

Виды наград:

1) Достижение определенного уровня

2) Некоторое количество созданных постов.

3) Высокий рейтинг постов.

Когда пользователь набирает определенную статистику, ему выдаётся награда.

Общие требования:
-------------------

Доступ к базе реализуется через EF (а не через SQL-запросы).

Проект должен быть выполнен аккуратно и целостно. 

Проект должен выглядеть как завершенное, готовое приложение. 

Код должен быть оформлен в соответствии с гайдлайнами. 

Должно быть использовано максимум готовых решений.
